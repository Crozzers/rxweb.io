ReactiveFormConfig is used to configure date format, validation messages globally at the start of the application.

About ReactiveFormConfig

ReactiveFormConfig is used to globally set date format,validation messages whenever the application starts it will initialized automatically. we have to set "validationMessage" for setting global validation message ,"baseConfig" for setting dateformat globally and "internationalization" for setting dateformat for worldwide applications.

baseConfig

To set the date format throughtout the application for date validation check, the baseConfig of ReactiveFormConfig is a configuration if you set then the applied the value prameter 'value' in 'minDate' and 'maxDate' validation works accordingly otherwise it will take the default format of 'mm/dd/yyyy'.Suppose date format in your country is 'mm/dd/yy', Set the dateformat as "mdy" or if it is 'dd/mm/yy' Set the dateformat as "dmy" and the format will be set globally.

internationalization

Suppose You want the application to work worldwide and you want the date format to validated according to your region,In that case you can change internationalization dynamically if your system supports it.

Validation Messages

Apply global validation messages throughout the application, then configure the validation messages globaly, You must set message for every validator you use.

Password validation Messages

You can set multiple messages for password validation globally into the ReactiveFormConfig as below.

ErrorMessageBindingStrategy

To configure the error message strategy globally when you have same strategy of showing error messages throughout the application.

The by default errorMessageBindingStrategy is ErrorMessageBindingStrategy.none.

Based upon your need you can set the errorMessageBindingStrategy to these conditions :
  1. ErrorMessageBindingStrategy.none : There is no condition to bind the error message, The error messages will be bind as soon as the FormControl is invalid.
  2. ErrorMessageBindingStrategy.onSubmit : Whenever the RxFormGroup property submitted is true then the invalid FormControl error messages will be bind.
  3. ErrorMessageBindingStrategy.onDirty: Once the FormControl is dirty, after that invalid FormControl error messages will be bind accordingly.
  4. ErrorMessageBindingStrategy.onTouched: Once the FormControl is touched, after that invalid FormControl error messages will be bind accordingly.
  5. ErrorMessageBindingStrategy.onDirtyOrTouched: Once the FormControl is Dirty or Touched, after that invalid FormControl error messages will be bind accordingly.

allowDecimalSymbol

Suppose you want to set the same decimal symbol throughout the application, then you must configure that decimal symbol globally in ReactiveFormConfig in app.component.ts. For example, if you want to set "," as your separator instead of "." then you can configure it by using allowDecimalSymbol.

defaultValidationLocale

This will allow multilingual characters based upon the specified validation alphabet locale in alpha and alphanumeric validations. It can be set to Spanish, danish, french, german, russian. Here it is globally set to spanish for alpha and alphanumeric validations.
