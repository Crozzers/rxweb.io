Extend Your Ngx-Translate Code

Extensible guide of @ngx-translate/core to @rxweb/ngx-translate-extension.

Installation

npm install @rxweb/translate @rxweb/ngx-translate-extension

Import TranslateModule

Throughout application the TranslateModule will be imported from @rxweb/ngx-translate-extension.

Before
import {{'{'}}TranslateModule{{'}'}} from "@ngx-translate/core"
After
import {{'{'}}TranslateModule{{'}'}} from "@rxweb/ngx-translate-extension"

Lazy Load Modules

There is no need use forChild in child modules.

Before
imports: [ TranslateModule.forChild({{'{'}}...{{'}'}}) ]
After
imports: [ TranslateModule ]

Use TranslateService

All methods and properties is same as before, the only thing is you have to change the package name, while importing the TranslateService.

Before
import {{'{'}}TranslateService{{'}'}} from "@ngx-translate/core"
After
import {{'{'}}TranslateService{{'}'}} from "@rxweb/ngx-translate-extension"

Your Own Loader

Slight change in paramter value, You will get string as well as LoaderTranslationConfig object. As LoaderTranslationConfig object contains 'translationName', 'lang' and 'filePath'. Its not necessary all time you will get all properties values.
