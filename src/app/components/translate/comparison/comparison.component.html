Before deep dive into the features would like to share the reason for developing this package.

The core objective of developing this package is to code must be simple and readable, anyone can understand easily without looking into the multiple files. Instead of reinventing the wheel try to use as much as an available feature from the Angular and TypeScript.

  1. Zero handwritten code for configuration, the 'rxweb' CLI automatically sets the configuration while setup the translation.
  2. Zero configuration required on module level imports or service creation. Import the RxWebModule in the root module and that works throughout the application.
  3. We love design patterns/principles and with this package, you don't need to write any business logic in the template as templates are not meant for that. Use Angular Interpolation double curly bracket for traduction the text as per active language even though instance parameters required in the text. The parameterized / Conditional text will be automatically resolved without passing the additional object ðŸ˜Š. You may like it.
  4. As we are a developer and we are busy developing overembellish functionalities to the application. By considering this case we don't want you to write jumble of code of resolving the multilingual content. Everything will be managed by using the @translate decorator even those are Smart Component or Dumb (Page) Component.
  5. Strongly Typed translation property. We won't say anything more about this. Have a look at the video.
  6. Simple translation management through JSON file in a structured way.
  7. Our hunger won't be fulfilled by knowing the above features as there is a need where JSON file won't work like Lookup data which is coming from the backend server and those will be changed once the application language is changed. Don't worry we have an optimum solution for this by passing the custom service class to the '@asyncTranslate` decorator. This gives full control to the developer to load the content according to the application of business logic. But wait!, How can we subscribe to the language change event? No, you don't haftoo worry about that, the custom service class automatically initialized and method will be called once the application language will be changed.
  8. On-demand parameter based scoped translation can be managed through TranslationService.
  9. Flexibility to load the translation content Before resolving the page component/shared component as well as you can load the content with Angular Pre Loading Strategy.
  10. Numerous ways to change the language of the application By URL as well as programmable.
  11. We don't prefer to cache the object into the browser memory or load the all language content during load time of the page. Only those multilingual content will be loaded which is required. As we said we don't prefer to cache the object into the browser memory because CDN is available then why we have to cache the object. But still we have given a feature to cache the multilingual content into the memory.
  12. The most extensive feature which saves your time with this package is automatically generates the keys in the JSON files based upon the written code in the template and Optimize the JSON file after build the project. Optimization is not that a like minify the JSON data into one line but this analyse all the template files and check that how many properties are defined in the JSON but not used in the template those will be removed after firing the optimize command through rxweb cli.
